Handling error

在go里面，个人感觉有几个很有用的包很有用，下面就记录一下。 - use fmt.Errorf(), fmt.Fprintf(os.Stderr, "") or log.Fatalf() to back probagate the error to the caller - If using log.Fatalf() for returning an error, use function like log.SetPrefix() for formatting the log message - EOF error: define EOF using ‘var err := errors.New(“EOF”)’ and check the EOF error type by if err == io.EOF go推荐所有在中间过程产生的error都抛给caller，再链式的返回main函数，所有中间产生error都应该被看作是bug。
