Today’s blog post will be recording my issues on variable scope when using python.

Below is a kind of normal code snippet that we have used for a long time in our daily development:

def genTownName(foo):
    if township.find("f")!=-1:
        result = True
    print(result)

If you are like me come from the world of java, you might find that the code above makes no sense since the variable result is assigned inside the if statement which should be declared outside it in order to be accessed. But it turns out that the print statement will output the result as expected. This is due to the context of variable result is actually inside the whole function so we can access it outside the if statement.

However, let’s take a look at the following code:

def genTownName(foo):
    if township.find("f")!=-1:
        result = True
    return result

Whenever you call this method, the following error will occur:

UnboundLocalError: local variable 'result' referenced before assignment

Why?

Recall the context of the variable result is actually inside the function genTownName. So after we return the result to the outside scope, the context is destroyed therefore the python interpretor could not recognize the variable anymore.
