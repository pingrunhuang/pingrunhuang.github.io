Strategy pattern
ad
Oct 14, 2018
2 minutes read

The Strategy Pattern defines a family of algorithms, encapsulates each one, and makes them interchangeable. Strategy lets the algorithm vary independently from clients that use it.

This is quoted from the book called .

From my daily development experience, I encountered a lot of class end with impl or start with I. I know they are meant for implementation and interface. But how come they are becoming some kind of must have for every java project that I encountered. Until recently, I watched the video explaining the strategy pattern which clear all the doubt that I had. So I decided to write down the summary and also comparing to functional language like python.

The basic idea of

flavor composition than inheritence.

public interface IQuach{
    public void quach();
}
public interface IFly{
    public void fly();
}

For each specific duck, a concret implementation of fly should be put in the block.

public class Duck{
    IQuach q;
    IFly   f;
    public Duck(IQuach q, IFly f){
        this.q=q;
        this.f=f;
    }
    public void fly(){
        this.q.fly();
    }
}

public class JetFlyImpl implements IFly{
    public void fly(){
        // jet fly algo
    }
}

This way, we can change the algo of quach method and fly method in a flexable way. I can just add more different algo for the same method withou changing the client code.
