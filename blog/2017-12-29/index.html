Don’t be constraint by a certain thinking pattern

I am a software engineer and recently I’m working on the data warehouse stuff. Data warehouse including data ETL(extract, transform, loading) is a very important chain in the big data stack. Most of the tools that I used is mysql, hive and spark. Most of the jobs for this part is to make sure data are flowing right. So the traditional relational database are very critical such as join table. In the past few weeks, I have to generate a big table from other tables given the certain information. I was first constrained by generating a dataframe to insert the whole into mysql database which turns out to be a very resource consuming solution. Since one of them are quite large and making the join manipulation is quite unnecessary. Inserting the recording one by might still be a good option although it seems like this is quite slow at the first glance.
