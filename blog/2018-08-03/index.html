what is the difference between checkpoint and persist(DISK_ONLY)?

Since checkpointand persist(DISK_ONLY) are both persisting RDD into disk. But what is the difference between these two? I think there are 2 main differences: 1. checkpoint will not persist the lineage of the DAG while the persist(DISK_ONLY) will persist the partitions of RDD and also the lineage. This difference make persist very good for other transformation that only require part of the transformation lineage. 2. when persist(DISK_ONLY) is called, the spark application will actually ask for the blockmanager to remember the info about the persisted RDD so that everytime when a job is called, it will ask the blockmanager if the RDD is already cached on disk for faster computation. checkpoint will only persist the result of the RDD and persist on external file system like HDFS or S3. Think of it as storing a temperary table on disk for further query. In which case, other spark application can also access it.

import org.apache.spark.storage.StorageLevel
