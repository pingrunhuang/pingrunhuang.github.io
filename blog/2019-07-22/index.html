BacNet protocol stands for building automation control network.

Why(aka where) do we need it?

Traditionally, different devices have their own way talking to each other. In order to let different devices to communicate with each other, a standard protocol is required which is why BacNet comes into play.

How to use that?

Different kind of server that I have encountered

  • nginx, tomcat, bacnet

Different componet requirements

Bacnet server:

  1. return 255 analog input (ID)
  2. need a server for querying data from redis and assemble the bacnet object to send back to the client.(checkout binaryio.py on how it create a bacNet server to run on RPI)
  3. get command from message queue

Bacnet client:

  1. Each time when a user touch the button of the switchpucks send to message queue
