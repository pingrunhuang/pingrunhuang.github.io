What docker volumes can do
ad
Feb 21, 2018
One minute read

I recently have to build a model server project with docker. The reason why I used docker is I want to separate the model server, rest api and the redis service. The python scripts that are used in the model server layer for generating model should be loaded as part of the container since this container is a service for serving model querying which require write locally and test virtually. So I have to use the volume concept in docker to load the local scripts.

Commands

  • docker run -v <dir-in-container> <container-name>: this will mount the dir-in-container to the host container directory in /var/lib/docker/volumes/ by default.
  • docker run --volumes-from <container-name>: this command will mount the volume in one container to the currently spawning container.
