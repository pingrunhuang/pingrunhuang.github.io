Ambari plugin development

What is ambari?

Ambari is a provisioning platform for cluster management which allow server administrator to check the health situation of the cluster and make some action accordingly from the web UI. Comparing to the traditional way of managing machines in the cluster by using bash script, this way is much more direct and more uniformed.

For example, when it comes to deploy a big-data application like ScaledRisk, it is very complicated and every person who deploy the application on their cluster might use different configuration parameters. This will cause a problem. Say our product is sent to our client who has zero knowledge about cluster stuff not even to say bash script. In this case, ambari provide the client with a friendly web UI to input different kind of parameters and the standard way to apply this parameters in order to get the application work.

Ambari plugin

Ambari provide plugins for services in hadoop eco system by default. It is also possible for service provider to design and add plugin into ambari.

Ambari python library

Ambari plugin is writen in python, which is inside the resource_management package mainly. Since the official ambari python library has not published yet, I tried to discover the basic classes and functionality of this library and finally make one for my purpose.

  • Exucute

    I think this is the major class in the python library. It is used as an execution of bash command for ambari.

Here, we provided and optional argument ‘logoutput’ and set it equals to True means we want to show the output of this command and log it into log file (which is located in /var/log/ambari-server/ambari-server.log). There are more argument such as user to specify the user who execute this command .etc.

  • Logger

    This class is used for logging logs to the logs file. There are different level of logging too. All are static methods. For example, to log an info, just use Logger.info('message'); to log an error, just use Logger.error('message') .etc.
