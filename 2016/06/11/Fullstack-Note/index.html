Fullstack note

Fullstacks fundamental

I recently took a basic course talking about fullstack development with a web app using flask and SQLAlchemy in python. I decided to take some note here. The course taught by creating a restaurant menu application.

Database layer

ORM(Object Relational Mapping) is a tool to convert code from python syntax into sql statement. Here we use SQLAlchemy to achieve our goal.

Database code is devided into 4 parts:

Configuration: import all required modules and dependencies

Class: represent data in python

Table: represent table in database

Mapper: map the column in database to python class

Check out the code following on how to implement them:
database_setup.py

python web server

Since we have the database working, we need to add the server engine in order to process the request from the client.

Difference between GET and POST:

Both are HTTP verb. GET simply get the information from server aka safe mode. If success, respond with status code 200.
POST ask to add, remove and change data on server. If success, respond with status code 301.

template engine in flask

Embedding html code in python application like we do previously is not very handy though. Flask provide a template engine to let python application embed the variable in an external html file and load the external html file by calling the render_template() method inside render_template module. This way, we can call python code inside html file(like jsp).
Using template engine, we can render the content of html just by using 2 lines of code instead of the previous several lines of code. In the above, we specify the html file called menu.html which should be put in the directory called templates. See how to manage the directory

url building

Flask also provide a function called [url_for()!](http://flask.pocoo.org/docs/0.10/quickstart/#url-building) which allows you to jump to other page to do other stuff. For example, we can change the for loop part in our html code shown before to add delete and edit function.
So when the client click on the button, the client will be redirect to a function inside the python code to handle the redirection.
Here is the example of how to use template in our menu application

handle post request

By default, flask route(url) method just handles get request which is not enough for updating database. To handle post request, we can specify inside the route() method by using route(url, method=['GET','POST']). Inside python code, we need to first import request, redirect and url_for module from flask. And then we need to write a method to handle post and get request.

Message flashing

Flask provide a simple way to prompt message to users. Message flashing is a feature that will prompt a message to user after several actions and disappear next time the page has been requested.
To use this feature, we need to import flash from flask module.

styling

Flask provide a way to add css file in the static file and add a reference link in the head of the page to load the css file.

responding with json

Sometimes we just want to check out the data instead of the html and css. In this case, we can use API to parse data to client. Flask also provide a way to build a API functionality called jsonify in the flask module. We need to first add serialize() function in database_setup.py file to tell Flask which part of object to be transfer through internet as JSON.
