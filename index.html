Design pattern -- Observer pattern

observer pattern

The Observer Pattern defines a one-to-many relationship between a set of objects.When the state of one object changes, all of its dependents are notified.
Purpose of this pattern is to minimize the coupling of different object.
Think it as a newspaper publisher and subscriber. The publisher (subject object) will send a newspaper to the subscriber (observer object) when there is an updated newspaper.
The observer pattern is either pull by the observer or push by the subject.This pattern is good for API development.
