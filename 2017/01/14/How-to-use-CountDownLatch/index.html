How to use CountDownLatch?

Recently I got a task to upgrade our existing application from java 7 to java 8. Which let me have an opportunity to explore more in the newer version of java.

From the documentation of the CountDownLatch, there are some points that worth pointing out:

  • The CountDownLatch object is initialized with the count with 1 and not changeable (use CyclicBarrier if you want to change the count)
  • The await() and countdown() methods are the most noticeable method. await() wait until the count member turned to 0 due to the invocation of countdown() method or the thread get interrupted.
  • The core data structure under the hood of CountDownLatch is using the AbstractQueuedSynchronizer aka AQS which will be discussed later.

So basically the CountDownLatch is like the latch on your door. Thinking the count variable inside CountDownLatch is the number of thread that is going to do things before the current thread who invoke await() method can continue.

The basic workflow of CountDownLatch might be something like this:

invoking await() -> lock the current thread -> invoke the countdown() method in somewhere else -> the previous thread that invoke await get continued due to the count become 0.
