Design pattern -- Factory pattern

Scenario

Pizza pizza = new CheesePizza() we want to create a new CheesePizza with the type implementing Pizza interface or abstract class here. But what if we want to instantiate another kind of pizza? Clearly, dealing with which concrete class is instantiated is really messing up our code and preventing it from being closed for modification.

Purpose

The Factory pattern provide an option to encapsulate the creation of an object by letting subclass decide what objects to create.

Use case

Code

Imagine you want to order a pizza from a pizza shop. There are New York pizza shop and Chicago pizza shop. Inside the pizza shop, there are different types of pizza.

abstract class

concrete implementation

summary

Factory pattern require a factory class to generate different kind of objects that share the same interface. Meanwhile, the factory class inherits a superclass which will have an abstract method used to be implemented by a certain factory class.
