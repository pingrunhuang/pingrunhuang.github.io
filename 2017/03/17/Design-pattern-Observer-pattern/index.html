Design pattern -- Observer pattern

observer pattern

The Observer Pattern defines a one-to-many relationship between a set of objects.When the state of one object changes, all of its dependents are notified.
Purpose of this pattern is to minimize the coupling of different object.
Think it as a newspaper publisher and subscriber. The publisher (subject object) will send a newspaper to the subscriber (observer object) when there is an updated newspaper.
The observer pattern is either pull by the observer or push by the subject.This pattern is good for API development.

Scenario

A weather station want to provide the weather data for several applications in order to let them present the weather info such as temperature, humidity and pressure etc. In this case, the weather station is like a subject (publisher) whereas the applications are the observer (subscriber).

Notice we are using the pull model

interface

concrete implementation

java’s built in Observer pattern

java.util.Observable: this is a class. Class that inherit this class is the subject
the Observable class contains a vector of observers so we do not need to write our own list
java.util.Observer: this is an interface. Class that implements this interface is the observer
Notice: the changed flag inside the Observable class is going to indicate if the state of the subject changed or not. There are some related method used to change this state which are setChanged() to set the change state to true, clearChanged() to set the state to false and hasChanged() to show the state.
