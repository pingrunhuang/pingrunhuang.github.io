How to use smartctl

In the scenario of heavily used cluster, the disks of the server are tend to be down for no reason. So it would be a good idea to locate which disks are tend to be down and replace those dangerous disks.

Environment

  • Operating system: CentOS 6.8
  • Server model: HP Proliant DL380 GS

Test procedure

In my case, my mount point are in the /dev/cciss/* (cciss represent for HP Smart Array block driver). So I have to add -d cciss,N at the end where N represent for the number of the partition. For example, if I have 8 partition of each disk, I have to loop through 8 partition for stress testing.
