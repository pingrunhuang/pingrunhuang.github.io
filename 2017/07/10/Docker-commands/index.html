Docker commands

Start a Container

  1. docker run --name <container-name> -p <local-address>:<local-port>:<container-port> -d image-name: -d means daemon mode, -p is linking the port of the host that run the docker(cloud or local) to the container port, --name is naming the container
  2. docker port <container-name> port: dynamically assign the local port linking to the port of the container.
  3. docker run --link <container-name>:<container-alias> -t image-name:version <start-shell>: run and link a container to another container. The version of the -t is optional, latest by default. This is good for starting a web container which link to a database container for safe connection.

Key to write a DockerFile

docker build -f <name-of-dockerfile> .: build the docker file from the current directory which contains the DockerFile.
docker build -t repository-name:tag .: specify a repository and tag to save the new image if build success.

After finishing the Dockerfile, we use sudo docker build -t image-name:version <context-directory>
